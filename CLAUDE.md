# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a **BMAD (Build Measure Analyze Deploy) v6.6.0** workspace called `table_project`. It uses the BMAD methodology framework for structured software project planning and development. The project is in early stages — artifacts and source code are generated through BMAD workflows.

- **User**: Lem (intermediate skill level)
- **Communication language**: Vietnamese
- **Document output language**: English

## Directory Structure

- `_bmad/` — BMAD framework config (installer-managed, treat as read-only). Customizations go in `_bmad/custom/`.
- `_out_put/planning-artifacts/` — PRDs, architecture docs, epics, stories generated by BMAD planning skills.
- `_out_put/implementation-artifacts/` — Code artifacts, test plans generated during development.
- `docs/` — Project knowledge base referenced by BMAD skills.
- `.claude/skills/` — BMAD skill definitions for Claude Code (mirrored in `.agents/skills/` for Codex).

## BMAD Workflow

The project follows a structured planning-to-implementation pipeline. Key skill groups in typical order:

1. **Discovery**: `/bmad-product-brief`, `/bmad-prfaq`, `/bmad-domain-research`, `/bmad-market-research`
2. **Requirements**: `/bmad-create-prd`, `/bmad-validate-prd`, `/bmad-edit-prd`
3. **Design**: `/bmad-create-ux-design`, `/bmad-create-architecture`
4. **Planning**: `/bmad-create-epics-and-stories`, `/bmad-create-story`, `/bmad-sprint-planning`
5. **Development**: `/bmad-dev-story`, `/bmad-quick-dev`, `/bmad-code-review`
6. **Quality**: `/bmad-qa-generate-e2e-tests`, `/bmad-review-adversarial-general`, `/bmad-review-edge-case-hunter`

Use `/bmad-help` to see available skills and guidance. Use `/bmad-check-implementation-readiness` before starting development to validate that planning artifacts are complete.

## Configuration

- Main config: `_bmad/config.toml` (read-only, regenerated by installer)
- User config: `_bmad/config.user.toml` (read-only, regenerated by installer)
- Custom overrides: `_bmad/custom/config.toml` (team) and `_bmad/custom/config.user.toml` (personal, gitignored)
- Use `/bmad-customize` to modify skill behavior.
